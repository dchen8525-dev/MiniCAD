# MiniCAD STEP 实体推荐实施顺序表

> 基于 `doc/minicad_step_entity_gap_report.md` 进一步压缩成可执行工作包，按“共享改动面 + 验证方式”排序。

## 说明

- 这里不再按 156 个实体逐个展开，而是合并成建议的实施工作包。
- `主改文件` 是第一落点，不代表只改这些文件。
- `完成标准` 优先要求：resolver 可解析、builder 可构建、preview/mesh 可导出、测试可覆盖。

## 实施原则

- 先做能形成端到端闭环的几何/拓扑工作包，再做只影响元数据可见性的工作包。
- 每个工作包至少同时补三类测试：resolver、builder 或 exporter、失败路径。
- 对“已解析未落地”的实体，优先补 `StepMetadataExtractor` / `ProductMetadataExtractor`，不要一上来就扩张几何 builder。
- 对“部分支持”的实体，先收敛限制和错误信息，再提升几何正确性；避免 silently wrong。
